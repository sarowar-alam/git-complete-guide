# Labs

Ten hands-on labs, each building on the last. Open Git Bash and work through them in order.

| Lab | Topic | Time | Internet needed? |
|-----|-------|------|-----------------|
| [lab-01](lab-01-version-control/README.md) | Your first Git repository | ~20 min | No |
| [lab-02](lab-02-branching/README.md) | Branching — create, switch, backup, merge | ~30 min | No |
| [lab-03](lab-03-merge-conflicts/README.md) | Merge conflicts & resolution | ~25 min | No |
| [lab-04](lab-04-rebase/README.md) | Rebase a feature branch | ~25 min | No |
| [lab-05](lab-05-pull-requests/README.md) | Fork → upstream sync → pull request | ~40 min | Yes (GitHub) |
| [lab-06](lab-06-branch-protection/README.md) | Branch protection rules | ~30 min | Yes (GitHub) |
| [lab-07](lab-07-tags-releases/README.md) | Tags & GitHub releases | ~25 min | Yes (GitHub) |
| [lab-08](lab-08-stash-reset-revert/README.md) | Stash, reset, revert | ~30 min | No |
| [lab-09](lab-09-history-recovery/README.md) | Reflog & bisect | ~30 min | No |
| [lab-10](lab-10-deployment/README.md) | Deploy to Render | ~35 min | Yes (GitHub + Render) |

---

Each lab follows this structure:
1. Objective
2. Architecture Diagram
3. Prerequisites
4. Setup
5. Step-by-Step Tasks
6. Validation
7. Expected Output
8. Troubleshooting
9. Cleanup
10. Challenge Task
